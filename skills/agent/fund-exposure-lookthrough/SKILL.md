# fund-exposure-lookthrough

Use this skill when working on or with the `fund-exposure-lookthrough` repository.

## Purpose

Build and verify static, broker-free fund exposure look-through reports from bundled or user-supplied CSV fixtures. Keep runtime behavior zero-dependency and public-safe.

## Workflow

1. Use `python -m fund_exposure_lookthrough.cli --help` to inspect available routes.
2. Generate artifacts with explicit roots, for example `python -m fund_exposure_lookthrough.cli build-packet --root .`.
3. Run `python -m fund_exposure_lookthrough.cli fixture-doctor --root .` to validate portfolio and constituent fixtures.
4. Run `python -m fund_exposure_lookthrough.cli policy-profile --root .`, `policy-compare --root . --left demo/policy_profile.json --right demo/policy_profile.json`, and `policy-gallery --root .` for research-only policy threshold artifacts.
5. Run `python -m fund_exposure_lookthrough.cli case-gallery --root .` and `reviewer-scorecard --root .` for public showcase artifacts.
6. Run `python -m fund_exposure_lookthrough.cli public-readiness --root .`, `docs-export --root .`, and `landing-page --root .` for the public landing and first-10-minute walkthrough layer.
7. Run `python -m fund_exposure_lookthrough.cli visual-receipt --root .`, `readme-snippet --root .`, `artifact-catalog --root .`, `command-matrix --root .`, `release-checklist --root .`, `final-handoff --root . --repo-url REPO_URL_PLACEHOLDER`, `maintainer-note --root .`, `final-qa --root . --baseline v0.5.0`, `asset-health --root .`, and `bundle-export --root .` for receipts, operator workflow, handoff, maintainer evidence, final QA, and audit artifacts.
8. Run `python -m fund_exposure_lookthrough.cli release-compare --root . --left demo/release_manifest.json --right demo/artifact_catalog.json`, `promotion-pack --root .`, and `adoption-notes --root .` for cross-run promotion evidence and thesis-ledger reuse notes.
9. Run `python -m fund_exposure_lookthrough.cli snapshot-ledger --root .`, `snapshot-compare --root . --left demo/snapshot_ledger.json --right demo/snapshot_ledger.json`, and `review-queue --root .` for state/history review.
10. Run `python -m fund_exposure_lookthrough.cli selfcheck --root .` before release.
11. Run `python -m pytest -q` when test tooling is available.

## Safety Rules

- Do not add live market data fetching.
- Do not connect to brokers or trading systems.
- Do not recommend buys, sells, holds, target allocations, or trades.
- Do not predict returns.
- Keep reports tied to user-supplied static inputs.
